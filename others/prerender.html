# 预渲染

# 什么是预渲染

预渲染就是将无法同步直出的页面（一般都是通过前端异步加载数据拼接DOM 的SPA路由），通过一些技术手段，获取访问对应路由生成好的html内容生成真实的文件，从而在实际上线之后，首次访问对应的路由就能够直接响应内容，而不是通过javascript动态的生成。

# 为啥要预渲染

  1. SEO
  2. 首屏响应速度

# 如何实现

# 第一步： prerender-spa-plugin

  1. prerender-spa-plugin安装

WARNING

prerender-spa-plugin依赖puppeteer,

而puppeteer依赖Chromium，Chromium文件较大 还需要翻墙。

// 跳过下载chromium
set PUPPETEER_SKIP_CHROMIUM_DOWNLOAD=1 && npm install prerender-spa-plugin
  1. 构建命令修改
    • 如果可以翻墙：手动下载chromium (需要翻墙)将下载的chromium 放到应用程序文件夹或者其他任意的文件夹下
    • 如无法翻墙 也可以使用google chrome 浏览器
    • 修改构建命令 (PUPPETEER_EXECUTABLE_PATH 就是你放置chromium或者google chrome的文件夹)
    //package.json
{
  "build": "PUPPETEER_EXECUTABLE_PATH=/Applications/Chromium.app/Contents/MacOS/Chromium vue-cli-service build"
}

    TIP

    PUPPETEER_EXECUTABLE_PATH 一定是可执行文件的全部路径 而不是存放执行文件的那个文件夹

  2. 修改webpack配置
  new PrerenderSPAPlugin({
      // Required - The path to the webpack-outputted app to prerender.
      staticDir: config.build.assetsRoot,
      // Required - Routes to render.
      routes: ["/manual", "/customer-service", "/lead"],
      renderer: new Renderer({
          inject: {
          },
          headless: false,
          renderAfterDocumentEvent: "render-event"
      })
  }),
  1. 修改main.js
const app = new Vue({
    router,
    store,
    mounted () {
    // You'll need this for renderAfterDocumentEvent.
        document.dispatchEvent(new Event("render-event"));
    },
    render: h=> h(App)
});
document.addEventListener("DOMContentLoaded", function () {
    app.$mount("#app");
});

详细的设置，请参考prerender-spa-plugin的文档。

# 第二步：配合vue-meta-info

在你预渲染的route根组件中添加SEO相关的设置

# 使用场景

  1. 非动态页面： 比如 关于我们、新手引导、客服等等。

# 注意事项

  1. 如果你的生成的线上包引用的静态资源需要引用线上资源地址，而在进行预渲染的时候没有发布 解决方案： 使用charles等代理工具代理到本地

# 和SSR服务器渲染的对比

  1. 解决的问题基本上相同：SEO 和首屏问题
  2. 预渲染是在构建的时候生成静态文件，服务器渲染是在接收到请求之后服务器端动态生成响应
  3. SSR 更加强大，相比预渲染成本更高（包括人力和机器成本）

具体请参考：vue ssr
